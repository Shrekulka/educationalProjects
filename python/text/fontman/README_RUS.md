# Описание проекта
## Название проекта: FontMan

## Цель проекта: 
FontMan предназначен для обработки документов Word, преобразуя текст в рукописный шрифт. Пользователи могут выбирать 
один из нескольких доступных шрифтов, после чего изменённый документ сохраняется под новым именем с добавлением 
приставки "updated" и суффикса даты.

## Задачи, которые решает проект:
1. Изменение шрифта документа: Позволяет преобразовать текст в документе в рукописный шрифт, делая его более 
   индивидуальным.
2. Управление шрифтами: Автоматически загружает и применяет шрифты, если они не установлены в системе, тем самым упрощая
   процесс работы пользователя.
3. Создание уникальных имён файлов: Формирует новое имя файла с добавлением временной метки, что позволяет избежать 
   перезаписи существующих документов.

## Особенности проекта:
- Использует локальные шрифты, если они не установлены в системе.
- Позволяет пользователю выбрать желаемый шрифт из доступного списка.
- Сохраняет обработанный документ в отдельной директории updated_documents.
- При отсутствии шрифта в системе программа временно регистрирует его с использованием библиотеки matplotlib, что
  позволяет избежать проблем при работе.

## Доступные шрифты:
### Программа предоставляет следующие шрифты для выбора:
1. Segoe Script
2. Brush Script MT
3. Mistral
4. Bradley's Pen
5. Ink Free
6. Выход - При выборе шести пользователю будет предложено завершить работу программы.

## Порядок исполнения кода:
1. Проверка аргументов командной строки: Программа принимает один аргумент - имя входного файла.
2. Создание директории для сохранения: Создаётся директория updated_documents, если она не существует.
3. Выбор шрифта: Пользователь выбирает шрифт из списка доступных рукописных шрифтов.
4. Загрузка шрифта: Если шрифт не установлен, программа загружает его из локальной директории.
5. Обработка документа: Применяется выбранный шрифт к тексту в документе.
6. Сохранение документа: Изменённый документ сохраняется в указанной директории с уникальным именем.

## Функции и их назначения:
1. main(): Основная функция, управляющая основным потоком выполнения программы.
2. load_fonts(font_name: str) -> None:
    - Загружает шрифт из локальных файлов, если он не установлен в системе.
    - Если шрифт уже установлен, функция завершает выполнение.
    - Если шрифт не установлен, она пытается загрузить его из указанной директории.
    - Аргументы:
        - font_name (str): Название шрифта, который нужно загрузить.
    - Возвращает: None.
3. process_document(input_file: str, output_file: str, font_choice: str):
    - Обрабатывает документ Word, изменяя стиль шрифтов на выбранный рукописный шрифт.
    - Аргументы:
        - input_file (str): Путь к входному файлу документа.
        - output_file (str): Путь к выходному файлу, куда будет сохранён обработанный документ.
        - font_choice (str): Выбранный шрифт для преобразования текста.
    - Возвращает: None.
4. get_handwriting_fonts() -> dict[str, str]:
    - Возвращает словарь доступных рукописных шрифтов, где ключами являются идентификаторы шрифтов, а значениями - их 
      названия.
    - Возвращает: dict[str, str]: Словарь доступных рукописных шрифтов.
5. get_font_file(font_name: str) -> str:
    - Возвращает имя файла шрифта по его названию.
    - Функция ищет соответствие между названием шрифта и его файловым именем.
    - Аргументы:
        - font_name (str): Название шрифта, для которого нужно получить имя файла.
    - Возвращает: str: Полный путь к файлу шрифта, если найден, иначе пустую строку.
6. is_font_installed(font_name: str) -> bool:
    - Проверяет, установлен ли шрифт в системе.
    - Функция пытается найти шрифт по его названию и возвращает результат проверки.
    - Аргументы:
        - font_name (str): Название шрифта, который нужно проверить.
    - Возвращает: bool: True, если шрифт установлен, иначе False.
7. get_font_path(font_name: str) -> str:
    - Возвращает путь к шрифту (локальному или установленному).
    - Функция сначала проверяет, установлен ли шрифт в системе.
    - Если установлен, возвращает его имя, иначе пытается найти его локальный файл.
    - Аргументы:
        - font_name (str): Название шрифта, для которого нужно получить путь.
    - Возвращает: str: Путь к шрифту. Если шрифт не найден, возвращает имя шрифта по умолчанию ('Calibri').

## Способы запуска и использования кода:
1. Через командную строку:
   Для запуска программы из командной строки используйте следующую команду:
    ```bash
    python fontman.py <имя_файла>
    ```
    Например:
    ```bash
    python fontman.py my_self.docx
    ```

2. Используя скрипт:
   Также можно запустить программу, используя предоставленный shell-скрипт run_fontman.sh, который автоматически 
   вызывает fontman.py с указанным файлом:
    ```bash
    ./run_fontman.sh <имя_файла>
    ```
    Например:
    ```bash
    ./run_fontman.sh my_self.docx
    ```

3. Из собранного дистрибутива:
   После сборки проекта с помощью `pyinstaller` можем запустить исполняемый файл из директории `dist/`, что позволяет 
   использовать программу без установки дополнительных зависимостей:
    ```bash
    cd dist/fontman
    ./fontman <имя_файла>
    ```
    Например:
    ```bash
    ./fontman my_self.docx
    ```
#### Аргументы:
<имя_файла>: Имя входного файла (документа Word), который будет обработан.


### Сохранение и наименование файлов:
Изменённый документ сохраняется в директории updated_documents с именем, составленным по следующему шаблону:
```bash
updated_<имя_входного_файла>_<временная_метка>.docx
```
где временная метка - это текущая дата и время в формате Unix.

В проекте существует requirements.txt: Этот файл содержит список зависимостей проекта, необходимых для его работы. Он 
используется для установки всех необходимых библиотек через пакетный менеджер pip.
