# AUTO.RIA Toyota Monitor Bot

Telegram бот для автоматического мониторинга объявлений Toyota на AUTO.RIA. Отслеживает новые объявления, изменения цен 
и проданные автомобили.

## Основные возможности
1. 🔍 Мониторинг новых объявлений Toyota
2. 💰 Отслеживание изменений цен
3. ✅ Уведомления о проданных автомобилях
4. 🖼️ Загрузка фотографий с AUTO.RIA
5. 🔨 Интеграция с аукционами (Copart, IAAI)
6. ⚙️ Настраиваемые параметры поиска

## Параметры поиска по умолчанию
1. Марка: Toyota
2. Модель: Sequoia (можно изменить на Tundra или Land Cruiser)
3. Фильтр по импорту из США: Включен
4. Фильтр по участию в ДТП: Включен

## Технические параметры по умолчанию
1. Интервал проверки: 10 минут
2. Порог изменения цены для уведомления: $1
3. Таймаут при ошибках: 60 секунд
4. Максимальное количество попыток запроса: 3

## Архитектура решения
1. Инициализация и запуск бота (bot.py)
    a) Создается экземпляр бота с указанным токеном
    b) Инициализируется диспетчер для обработки сообщений
    c) Создаются сервисные классы:
        - **Database** - для работы с SQLite
        - **TelegramService** - для отправки сообщений
        - **CarService** - основная бизнес-логика

    d) Запускается поллинг для получения обновлений

2. Основной цикл работы (CarService)
    ```bash
    while self.is_running:
        await self.check_updates()
        await asyncio.sleep(check_interval)  # 600 секунд по умолчанию
    ```

    **В методе check_updates:**
    a) Получаем список пользователей из БД
    b) Для каждого пользователя:
        - Загружаем его настройки (модель, фильтры)
        - Получаем текущие объявления через парсер
        - Обрабатываем новые авто, изменения цен и проданные

## База данных и хранение данных

**Структура таблиц SQLite**
1. **cars:** основная информация об объявлениях
2. **price_history:** история изменения цен
3. **user_settings:** настройки пользователей

### Основные операции с БД
- Добавление новых авто
- Обновление цен
- Маркировка проданных
- Управление настройками пользователей

## Система уведомлений (TelegramService)
1. Новое объявление
    - Основная информация (название, цена, ссылка)
    - Альбом фотографий с AUTO.RIA
    - Если есть - альбом фото с аукциона

2. Изменение цены
    - Старая и новая цена
    - Разница с emoji-индикатором

3. Продажа автомобиля
    - Информация о проданном авто
    - Последняя известная цена

## Защита от блокировок и обработка ошибок
- Задержки между запросами
- Ротация прокси-серверов
- Повторные попытки при ошибках
- Имитация поведения браузера через User-Agent
- Логирование всех ошибок
- Graceful shutdown при остановке

## Оптимизация производительности
- Асинхронное выполнение запросов
- Переиспользование сессий aiohttp
- Кэширование настроек пользователей
- Автоматическое восстановление после сбоев

## Важные замечания по эксплуатации
- Бот использует HTML-разметку в сообщениях
- Для работы требуется доступ к API Telegram
- Необходимы права администратора в канале
- Рекомендуется использовать прокси-серверы
- База данных SQLite должна быть доступна для записи

## Решение проблем и рекомендации
Частые проблемы
- "Сообщение не изменено" - игнорируется
- Ошибки парсинга - повторная попытка
- Проблемы с фото - пропуск и логирование

## Рекомендации по обслуживанию
- Регулярно проверяйте логи
- Настройте уведомления об ошибках
- Используйте разные прокси-серверы
- Следите за лимитами Telegram API

## Оптимальные настройки
- Начните с мониторинга одной модели
- Установите разумный порог для уведомлений об изменении цены
- Используйте выделенный канал для уведомлений

## Процесс работы
1. При запуске бот проверяет наличие таблиц в базе данных
2. Каждые 10 минут выполняется:
    - Парсинг объявлений с AUTO.RIA
    - Проверка новых объявлений
    - Проверка изменений цен
    - Проверка проданных автомобилей
3. При обнаружении изменений:
    - Отправка уведомлений в канал
    - Обновление базы данных
    - Загрузка фотографий с аукционов (при наличии)
