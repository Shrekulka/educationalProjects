# Этот проект реализует двунаправленный потоковый gRPC-сервис для обмена сообщениями в реальном времени. 
# Проект использует Google Protocol Buffers (protobuf) для определения структуры сообщений и gRPC для обмена 
# этими сообщениями между клиентом и сервером.

## Основные компоненты проекта
- `bidirectional_streaming.proto:` Файл, определяющий протокол для gRPC-сервиса, включая сообщения чата и метод сервиса.
- `bidirectional_streaming_server.py:` Серверный скрипт, который реализует gRPC-сервис. Он обрабатывает двунаправленный 
   поток сообщений между клиентами и сервером.
- `bidirectional_streaming_pb2.py` и `bidirectional_streaming_pb2_grpc.py:` Сгенерированные файлы Python, необходимые 
   для работы с gRPC и protobuf. Эти файлы создаются на основе `bidirectional_streaming.proto`.

## Настройка проекта
Перед запуском сервера необходимо выполнить несколько шагов настройки:

*Установите зависимости:*
Для работы gRPC и protobuf необходимо установить соответствующие библиотеки. Выполните следующую команду, чтобы 
установить все зависимости:
```bash
pip install -r requirements.txt
```
*В этом проекте необходимые файлы уже были сгенерированы с помощью следующих команд:*
```bash
python -m grpc_tools.protoc --proto_path=. ./bidirectional_streaming.proto --python_out=. --grpc_python_out=.
```
*Разъяснения:* 
`--proto_path=.`: Указывает директорию, где находится файл .proto. 
`--python_out=.`: Указывает директорию, куда будут сохранены сгенерированные Python-файлы для сообщений protobuf. 
`--grpc_python_out=.`: Указывает директорию, куда будут сохранены сгенерированные Python-файлы для gRPC.

В результате этих шагов были созданы файлы `bidirectional_streaming_pb2.py` и `bidirectional_streaming_pb2_grpc.py`, на 
основе определения в `bidirectional_streaming.proto`. Эти файлы необходимы для работы gRPC-сервера и клиента.

## Что делает код
1. Файл `bidirectional_streaming.proto` определяет:
   - Сообщение ChatMessage с полями для имени пользователя, текста сообщения и временной метки.
   - gRPC-сервис ChatService с методом Chat, реализующим двунаправленный потоковый RPC.
2. Файл `bidirectional_streaming_server.py` реализует сервер:
   - Класс ChatServicer реализует метод Chat, который обеспечивает двунаправленный потоковый обмен сообщениями.
   - Метод serve() инициализирует gRPC-сервер, регистрирует ChatServicer и запускает сервер на порту 50054.
   
## Последовательность операций
1. Клиент отправляет сообщения в поток, используя метод Chat.
2. Сервер получает сообщения и добавляет их в общий список сообщений.
3. Сервер постоянно отправляет новые сообщения клиентам, которые подключены к сервису.
4. Клиенты получают сообщения в режиме реального времени.

## Взаимодействие с Postman
*Postman поддерживает тестирование gRPC-запросов. Вот как вы можете настроить и протестировать gRPC-сервис:*
1. Запустите gRPC-сервер:
    ```bash
    python bidirectional_streaming_server.py
    ```
2. Откройте Postman и создайте новый gRPC-запрос:
   - Нажмите на "New" и выберите "gRPC Request".
   - Введите адрес сервера: localhost:50054.
3. Выберите сервис и метод:
   - Postman автоматически загрузит доступные сервисы и методы, если включен reflection.
   - Выберите сервис ChatService и метод Chat.
4. Настройте поток запроса и ответа: 
   *Пример запроса:*
    ```bash
    {
      "user_name": "Alice",
      "message": "Hello, world!",
      "timestamp": 1692948482000
    }
    ```
5. Отправьте запрос и просмотрите ответ:
   - Нажмите "Send", чтобы отправить запрос на сервер.
   - В правой части окна Postman отобразится ответ сервера.
   
## Отличия между типами RPC

| Характеристика        | Унарный RPC                          | Серверный потоковый RPC                                       | Клиентский потоковый RPC                                   | Двунаправленный потоковый RPC        |
|-----------------------|--------------------------------------|---------------------------------------------------------------|------------------------------------------------------------|--------------------------------------|
| Направление данных    | Клиент → Сервер → Клиент             | Клиент → Сервер → Клиент (многократно)                        | Клиент (многократно) → Сервер → Клиент                     | Клиент ↔ Сервер                      |
| Количество запросов   | Один                                 | Один                                                          | Много                                                      | Много                                |
| Количество ответов    | Один                                 | Много                                                         | Один                                                       | Много                                |
| Пример использования  | Простые операции (напр., калькулятор)| Отправка больших объемов данных клиенту (напр., анализ строки)| Отправка потока данных серверу (напр., вычисление среднего)| Постоянный обмен данными (напр., чат)|
| Сложность реализации  | Низкая                               | Средняя                                                       | Средняя                                                    | Высокая                              |
| Управление состоянием | Простое                              | Среднее (на стороне сервера)                                  | Среднее (на стороне клиента)                               | Сложное (на обеих сторонах)          |
| Latency               | Низкая                               | Средняя                                                       | Средняя                                                    | Может быть низкой                    |
| Использование ресурсов| Низкое                               | Среднее                                                       | Среднее                                                    | Высокое                              |

## Заключение
Данный проект демонстрирует реализацию двунаправленного потокового gRPC-сервиса для обмена сообщениями в реальном 
времени. Он использует двунаправленный потоковый RPC, что делает его подходящим для чатов и других приложений, требующих 
постоянного обмена данными. Postman является удобным инструментом для тестирования gRPC-сервисов, благодаря поддержке 
gRPC-запросов и возможности динамического обнаружения сервисов.
