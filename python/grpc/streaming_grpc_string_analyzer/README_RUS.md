# Этот проект реализует gRPC-сервер и клиент для анализа строк в потоковом режиме. Сервер принимает строку для анализа и
# возвращает результаты анализа, используя gRPC потоки. Проект использует Google Protocol Buffers (protobuf) для 
# определения структуры сообщений и gRPC для обмена этими сообщениями между клиентом и сервером.

## Основные компоненты проекта
- `streaming.proto`: Файл, определяющий протокол для gRPC-сервиса, включая сообщения запроса и ответа, а также методы 
   сервиса.
- `streaming_server.py`: Серверный скрипт, который реализует gRPC-сервис. Он принимает строки на анализ и отправляет 
   обратно результаты в виде потоков данных.
- `streaming_pb2.py` и `streaming_pb2_grpc.py`: Сгенерированные файлы Python, необходимые для работы с gRPC и protobuf. 
   Эти файлы создаются на основе `streaming.proto`.

## Настройка проекта

Перед запуском сервера необходимо выполнить несколько шагов настройки:

*Установите зависимости:*
Для работы gRPC и protobuf необходимо установить соответствующие библиотеки. Выполните следующую команду, чтобы 
установить все зависимости:
```bash
pip install -r requirements.txt
```
*В этом проекте необходимые файлы уже были сгенерированы c помощью следующих команд:*
```bash
python -m grpc_tools.protoc --proto_path=. ./streaming.proto --python_out=. --grpc_python_out=.
```
*Разъяснения:*
`--proto_path=.`: Указывает директорию, где находится файл .proto.
`--python_out=.`: Указывает директорию, куда будут сохранены сгенерированные Python-файлы для сообщений protobuf.
`--grpc_python_out=.`: Указывает директорию, куда будут сохранены сгенерированные Python-файлы для gRPC.

В результате этих шагов будут созданы файлы `streaming_pb2.py` и `streaming_pb2_grpc.py`, на основе определения в 
`streaming.proto`. Эти файлы необходимы для работы gRPC-сервера и клиента.

## Что делает код
1. Файл `streaming.proto` определяет:
   - Сообщения StreamDataRequest и StreamDataResponse, которые описывают формат запросов и ответов.
   - gRPC-сервис StreamingService с методами потокового анализа строк.
2. Файл `streaming_server.py` реализует сервер:
   - Класс StringAnalyzerStreamingServicer реализует метод StreamData, который анализирует строку и отправляет поток 
     результатов.
   - Метод serve() инициализирует gRPC-сервер, регистрирует StringAnalyzerStreamingServicer и запускает сервер на порту 
     50052.

## Последовательность операций
1. Клиент отправляет запрос StreamDataRequest на сервер с входной строкой для анализа.
2. Сервер получает запрос и передает его в метод StreamData.
3. Метод StreamData выполняет анализ строки (проверка палиндромов, подсчет слов, букв, пробелов, спецсимволов, заглавных и строчных букв) и возвращает поток ответов StreamDataResponse.
4. Сервер отправляет результаты анализа клиенту через поток.

## Взаимодействие с Postman

*Postman поддерживает тестирование gRPC-запросов. Вот как вы можете настроить и протестировать gRPC-сервис:*
1. Запустите gRPC-сервер:
    ```bash
    python streaming_server.py
    ```
2. Откройте Postman и создайте новый gRPC-запрос:
   - Нажмите на "New" и выберите "gRPC Request".
   - Введите адрес сервера: localhost:50052.
3. Выберите сервис и метод:
   - Postman автоматически загрузит доступные сервисы и методы, если включен reflection.
   - Выберите сервис StreamingService и метод StreamData.
4. Настройте тело запроса: 
   *Пример запроса для анализа строки:*
    ```bash
    {
      "input_text": "Hello World"
    }
    ```
5. Отправьте запрос и просмотрите поток ответов:
   - Нажмите "Send", чтобы отправить запрос на сервер.
   - В правой части окна Postman отобразится поток ответов сервера.

## Отличия между типами RPC

| Характеристика        | Унарный RPC                          | Серверный потоковый RPC                                       | Клиентский потоковый RPC                                   | Двунаправленный потоковый RPC        |
|-----------------------|--------------------------------------|---------------------------------------------------------------|------------------------------------------------------------|--------------------------------------|
| Направление данных    | Клиент → Сервер → Клиент             | Клиент → Сервер → Клиент (многократно)                        | Клиент (многократно) → Сервер → Клиент                     | Клиент ↔ Сервер                      |
| Количество запросов   | Один                                 | Один                                                          | Много                                                      | Много                                |
| Количество ответов    | Один                                 | Много                                                         | Один                                                       | Много                                |
| Пример использования  | Простые операции (напр., калькулятор)| Отправка больших объемов данных клиенту (напр., анализ строки)| Отправка потока данных серверу (напр., вычисление среднего)| Постоянный обмен данными (напр., чат)|
| Сложность реализации  | Низкая                               | Средняя                                                       | Средняя                                                    | Высокая                              |
| Управление состоянием | Простое                              | Среднее (на стороне сервера)                                  | Среднее (на стороне клиента)                               | Сложное (на обеих сторонах)          |
| Latency               | Низкая                               | Средняя                                                       | Средняя                                                    | Может быть низкой                    |
| Использование ресурсов| Низкое                               | Среднее                                                       | Среднее                                                    | Высокое                              |

## Заключение
Данный проект демонстрирует реализацию gRPC-сервиса для потокового анализа строк. Использование потокового типа RPC 
делает его подходящим для задач, требующих последовательной обработки и передачи результатов. Postman является удобным 
инструментом для тестирования gRPC-сервисов, благодаря поддержке gRPC-запросов и возможности динамического обнаружения 
сервисов.
