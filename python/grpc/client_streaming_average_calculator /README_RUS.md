# Этот проект реализует gRPC-сервер и клиент для вычисления среднего значения из потока чисел, отправляемого клиентом. 
# Проект использует Google Protocol Buffers (protobuf) для определения структуры сообщений и gRPC для обмена этими 
# сообщениями между клиентом и сервером.

## Основные компоненты проекта
- `client_streaming.proto`: Протокольный файл, который определяет структуру сообщений и метод сервиса gRPC для вычисления
   среднего значения.
- `client_streaming_server.py`: Серверный скрипт, который реализует gRPC-сервис. Он принимает поток чисел, вычисляет их
   среднее значение и возвращает результат.
- `client_streaming_pb2.py` и `client_streaming_pb2_grpc.py`: Сгенерированные файлы Python, необходимые для работы с 
   gRPC и protobuf. Эти файлы создаются на основе client_streaming.proto.

## Настройка проекта

Перед запуском сервера необходимо выполнить несколько шагов настройки:

*Установите зависимости:*
Для работы gRPC и protobuf необходимо установить соответствующие библиотеки. Выполните следующую команду, чтобы 
установить все зависимости:
```bash
pip install -r requirements.txt
```
*В этом проекте необходимые файлы уже были сгенерированы c помощью следующих команд:*
```bash
python -m grpc_tools.protoc --proto_path=. ./client_streaming.proto --python_out=. --grpc_python_out=.
```
*Разъяснения:*
`--proto_path=.`: Указывает директорию, где находится файл .proto.
`--python_out=.`: Указывает директорию, куда будут сохранены сгенерированные Python-файлы для сообщений protobuf.
`--grpc_python_out=.`: Указывает директорию, куда будут сохранены сгенерированные Python-файлы для gRPC.

В результате этих шагов были созданы файлы `client_streaming_pb2.py` и `client_streaming_pb2_grpc.py`, на основе 
определения в `client_streaming.proto`. Эти файлы необходимы для работы gRPC-сервера и клиента.

## Что делает код
1. Файл `client_streaming.proto` определяет:
   - Сообщение NumberRequest для отправки чисел в потоке.
   - Сообщение AverageResponse, которое возвращает результат вычисления среднего значения.
   - gRPC-сервис AverageCalculator с клиентским потоковым методом CalculateAverage.
2. Файл `client_streaming_server.py` реализует сервер:
   - Класс AverageCalculatorServicer реализует метод CalculateAverage, который принимает поток чисел и вычисляет их 
     среднее значение.
   - Метод serve() инициализирует gRPC-сервер, регистрирует AverageCalculatorServicer и запускает сервер на порту 50053.
   
## Последовательность операций
1. Клиент отправляет поток запросов NumberRequest на сервер, содержащий числа.
2. Сервер принимает поток запросов и передает их в метод CalculateAverage.
3. Метод CalculateAverage вычисляет среднее значение из полученного потока чисел.
4. Сервер отправляет ответ AverageResponse клиенту, содержащий среднее значение.

## Взаимодействие с Postman

*Postman поддерживает тестирование gRPC-запросов, включая клиентские потоковые запросы. Вот как вы можете настроить и 
протестировать gRPC-сервис:*
1. Запустите gRPC-сервер:
    ```bash
    python client_streaming_server.py
    ```
2. Откройте Postman и создайте новый gRPC-запрос:
   - Нажмите на "New" и выберите "gRPC Request".
   - Введите адрес сервера: localhost:50053.
3. Выберите сервис и метод:
   - Postman автоматически загрузит доступные сервисы и методы, если включен reflection.
   - Выберите сервис AverageCalculator и метод CalculateAverage.
4. Настройте поток запросов:
   *Пример запроса для отправки нескольких чисел:*
   - С права, от выбранного метода, кнопку `Invoke`
   - В поле `Message` первый раз введите и нажмите на `send`:
    {
    "number": 10
    }
    - Второй раз введите и нажмите на `send`:
    {
    "number": 20
    }
    - Третий раз введите и нажмите на `send`:
    {
    "number": 30
    }
    - Четвертый раз введите и нажмите на `send`:
    {
    "number": 40
    }
    - Например, на пятый раз, нажмите на кнопку (рядом с `send`) `End Streaming`:

5. В правой части окна Postman отобразится ответ сервера со средним значением.

## Отличия между типами RPC

| Характеристика        | Унарный RPC                          | Серверный потоковый RPC                                       | Клиентский потоковый RPC                                   | Двунаправленный потоковый RPC        |
|-----------------------|--------------------------------------|---------------------------------------------------------------|------------------------------------------------------------|--------------------------------------|
| Направление данных    | Клиент → Сервер → Клиент             | Клиент → Сервер → Клиент (многократно)                        | Клиент (многократно) → Сервер → Клиент                     | Клиент ↔ Сервер                      |
| Количество запросов   | Один                                 | Один                                                          | Много                                                      | Много                                |
| Количество ответов    | Один                                 | Много                                                         | Один                                                       | Много                                |
| Пример использования  | Простые операции (напр., калькулятор)| Отправка больших объемов данных клиенту (напр., анализ строки)| Отправка потока данных серверу (напр., вычисление среднего)| Постоянный обмен данными (напр., чат)|
| Сложность реализации  | Низкая                               | Средняя                                                       | Средняя                                                    | Высокая                              |
| Управление состоянием | Простое                              | Среднее (на стороне сервера)                                  | Среднее (на стороне клиента)                               | Сложное (на обеих сторонах)          |
| Latency               | Низкая                               | Средняя                                                       | Средняя                                                    | Может быть низкой                    |
| Использование ресурсов| Низкое                               | Среднее                                                       | Среднее                                                    | Высокое                              |

## Заключение
Этот проект демонстрирует реализацию клиентского потокового gRPC-сервиса для вычисления среднего значения из потока 
чисел. Клиентский потоковый RPC позволяет отправлять непрерывный поток данных с клиента на сервер, что делает его 
подходящим для задач, требующих агрегации данных на серверной стороне перед отправкой окончательного ответа. 
Использование Postman для тестирования gRPC-сервисов облегчает процесс отладки и проверки работы сервиса.
