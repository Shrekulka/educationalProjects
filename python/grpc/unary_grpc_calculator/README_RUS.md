# Этот проект реализует простой gRPC-сервер для выполнения основных арифметических операций, таких как сложение,
# вычитание, умножение и деление. Проект использует Google Protocol Buffers (protobuf) для определения структуры сообщений
# и gRPC для обмена этими сообщениями между клиентом и сервером.

## Основные компоненты проекта
- `unary.proto:` Файл, определяющий протокол для gRPC-сервиса, включая сообщения запроса и ответа, а также методы 
  сервиса.
- `unary_server.py:` Серверный скрипт, который реализует gRPC-сервис. Он принимает запросы на выполнение арифметических
  операций и отправляет обратно результаты.
- `unary_pb2.py` и `unary_pb2_grpc.py:` Сгенерированные файлы Python, необходимые для работы с gRPC и protobuf. Эти 
  файлы создаются на основе unary.proto.

## Настройка проекта
Перед запуском сервера необходимо выполнить несколько шагов настройки:

*Установите зависимости:*
Для работы gRPC и protobuf необходимо установить соответствующие библиотеки. Выполните следующую команду, чтобы 
установить все зависимости:
```bash
pip install -r requirements.txt
```

*В этом проекте необходимые файлы уже были сгенерированы c помощью следующих команд:*
```bash
python -m grpc_tools.protoc --proto_path=. ./unary.proto --python_out=. --grpc_python_out=.
```
*Разьяснения:*
`--proto_path=.`: Указывает директорию, где находится файл .proto.
`--python_out=.`: Указывает директорию, куда будут сохранены сгенерированные Python-файлы для сообщений protobuf.
`--grpc_python_out=.`: Указывает директорию, куда будут сохранены сгенерированные Python-файлы для gRPC.

В результате этих шагов были созданы файлы `unary_pb2.py` и `unary_pb2_grpc.py`, на основе определения в `unary.proto`. 
Эти файлы необходимы для работы gRPC-сервера и клиента.

## Что делает код

1. Файл `unary.proto` определяет:
   - Перечисление Operation с типами операций: ADD, SUBTRACT, MULTIPLY, DIVIDE.
   - Сообщения CalculationRequest и CalculationResponse, которые описывают формат запросов и ответов.
   - gRPC-сервис Calculator с унарным методом Calculate.
   
2. Файл `unary_server.py` реализует сервер:
   - Класс CalculatorServicer реализует метод Calculate, который выполняет арифметическую операцию, переданную в запросе.
   - Метод serve() инициализирует gRPC-сервер, регистрирует CalculatorServicer и запускает сервер на порту 50051.
   
## Последовательность операций
1. Клиент отправляет запрос CalculationRequest на сервер с двумя числами и типом операции.
2. Сервер получает запрос и передает его в метод Calculate.
3. Метод Calculate выполняет операцию (сложение, вычитание, умножение или деление) и формирует ответ CalculationResponse.
4. Сервер отправляет ответ клиенту.

### Взаимодействие с Postman

*Postman поддерживает тестирование gRPC-запросов. Вот как вы можете настроить и протестировать gRPC-сервис:*
1. Запустите gRPC-сервер:
    ```bash
    python unary_server.py
    ```
2. Откройте Postman и создайте новый gRPC-запрос:
   - Нажмите на "New" и выберите "gRPC Request".
   - Введите адрес сервера: localhost:50051.
3. Выберите сервис и метод:
   - Postman автоматически загрузит доступные сервисы и методы, если включен reflection.
   - Выберите сервис Calculator и метод Calculate.
4. Настройте тело запроса:
   *Пример запроса для операции сложения:*
    ```bash
    {
      "num1": 10,
      "num2": 5,
      "operation": "ADD"
    }
    ```
5. Отправьте запрос и просмотрите ответ:
   - Нажмите "Send", чтобы отправить запрос на сервер.
   - В правой части окна Postman отобразится ответ сервера.
   
## Отличия между типами RPC
| Характеристика              | Унарный RPC              | Серверный потоковый RPC         | Клиентский потоковый RPC         | Двунаправленный потоковый RPC     |
|-----------------------------|--------------------------|---------------------------------|----------------------------------|-----------------------------------|
| **Определение в proto**     | Один запрос, один ответ  | Один запрос, поток ответов      | Поток запросов, один ответ       | Поток запросов, поток ответов     |
| **Ключевое слово `stream`** | Не используется          | Используется для ответа         | Используется для запроса         | Используется для запроса и ответа |
| **Метод сервера**           | Обычная функция          | Использует `yield`              | Принимает итератор               | Принимает и возвращает итератор   |
| **Обработка данных**        | Одноразовая              | Последовательная отправка       | Агрегация входящих данных        | Интерактивный обмен               |
| **Применение**              | Простые запросы          | Отправка больших объемов данных | Отправка потока данных на сервер | Чат, реальный обмен               |

## Заключение
Данный проект демонстрирует реализацию базового gRPC-сервиса для выполнения арифметических операций. Он использует 
унарный тип RPC, что делает его подходящим для простых запросов и ответов. Postman является удобным инструментом для 
тестирования gRPC-сервисов, благодаря поддержке gRPC-запросов и возможности динамического обнаружения сервисов.
